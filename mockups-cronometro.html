⚡ Bocetos Cronómetro — Ejercicios Multi-Intensidad

4 opciones para resolver: descanso entre intensidades, preview del siguiente peso, y agrupación visual.
Contexto: Arranque de Fuerza con bloques 70% → 80% → 85%.

Opción A — Auto-Rest entre mismos ejercicios

Cambio mínimo. Cuando el siguiente ejercicio comparte el mismo ID base, en vez de pausar con "Ejercicio Completo", se auto-continúa con un descanso + banner de preview.

  • Los ejercicios siguen siendo entradas separadas en el listado
  • El contador sigue mostrando "Ejercicio 2 de 8"
  • Se agrega un banner "SIGUIENTE" durante el rest
  • Auto-rest en vez de pausa completa entre intensidades

Opción B — Agrupación Visual

Opción A + agrupación. Los ejercicios con mismo ID base se muestran como "bloques" de un mismo ejercicio. El contador dice "Bloque 2/3" en vez de "Ejercicio X de Y".

  • Auto-rest entre intensidades (igual que A)
  • Banner de preview (igual que A)
  • Contador de bloque: "Arranque de Fuerza — Bloque 2/3 (80%)"
  • En el listado idle, sub-items agrupados

Opción C — Series con Datos Variables por Ronda

Refactor mayor del modelo de datos. Cada ejercicio tiene un array de series con kg/reps/pct individuales. El card muestra la serie actual y la siguiente.

  • Un solo ejercicio "Arranque de Fuerza" con 8 series totales (3+3+2)
  • Cada serie tiene su propio kg y reps
  • El card muestra la serie actual + mini-preview de la siguiente
  • Requiere cambios en types, reducer, extractTimerExercises, display

Opción D — Rueda / Carrusel (estilo iPhone Picker)

Las intensidades del mismo ejercicio se muestran como un carrusel vertical tipo rueda de iPhone. La intensidad actual está centrada, grande y nítida. La anterior aparece arriba más pequeña y opaca. La siguiente abajo, también reducida. Es como un scroll infinito que gira.

  • El foco está siempre en la intensidad actual
  • Preview natural: el atleta VE el siguiente peso sin necesidad de un banner separado
  • La ya completada queda arriba como referencia (con check verde)
  • Puede combinarse con auto-rest (Opción A) para el flujo entre cards
